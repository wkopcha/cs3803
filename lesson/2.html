Creating and Using Threads in Java

In the previous lesson, you have learned about Multithreading and the difference between processes and threads. In this lesson, we will cover how to create and use Threads in Java.

There are two ways to create a Thread in Java:

  • Extending the Thread class
  • Implementing the Runnable interface

Extending the Thread Class

Let’s consider an example below when extending the Thread class.

public class MyThread extends Thread {
    // We write this method in order to start the thread
    public void run() {
        System.out.println("Thread Started Running!");
    }
    public static void main(String[] args) {
        MyThread thread = new MyThread();

        // Starting the thread
        thread.start();
    }
}

Output:

Thread Started Running!

Explanation:

In the code snippet above, we extend the Thread class which is contained in the built-in java.lang package. We create a run() method for the Thread to start executing desired tasks, and then we invoke the Thread by instantiating the Thread object and calling the start() method on this object.

Implementing the Runnable Interface

An alternative way to create a Thread is to implement the Java Runnable interface. Consider the following example.

public class MyThread2 implements Runnable {
    // Required to implement this method to start the Thread
    public void run() {
        System.out.println(
                "Thread created by implementing Runnable is running!");
    }

    public static void main(String[] args) {
        MyThread2 thread = new MyThread2();
        // creating the Thread Object
        Thread t1 = new Thread(thread);
        t1.start();
    }
}

Output

Thread created by implementing Runnable is running!

Explanation

Instead of extending the Thread class, we implement the Runnable interface. Similarly, we define a run method inside the MyThread2 class. During initialization, we first instantiate the custom thread MyThread2 we created into the ‘thread’ variable and then use the built-in Thread class constructor in which we pass in the ‘thread’ variable to create a Thread. Finally, we run it as in the previous example using the Thread class ‘start’ method.

Comparing Two Ways to Create a Thread

"When a class extends the Thread class, it can't extend any other class due to Java's single inheritance limitation. However, by implementing the Runnable interface, it's possible to extend from another class as well. For example, you can have a class like MyClass that extends another class (e.g., OtherClass) and implements the Runnable interface: class MyClass extends OtherClass implements Runnable."

Concurrency problem? - Lack of knowledge in which order the code will run.

One of the challenges with multithreading is the unpredictability of the execution order of threads. This is known as a concurrency problem. Without proper management, it can lead to issues such as race conditions, where multiple threads try to modify the same resource simultaneously, leading to unpredictable outcomes.
